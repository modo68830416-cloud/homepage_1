# 사용자 가치 제안

- 작업 ID: TASK-001
- 관련 문서: [[project-vision]]

## 1. 사용자 약속 (Commitments)

| 약속 | 구체적 의미 | 위반으로 간주하는 사례 |
|---|---|---|
| 이해하기 쉬운 정보 | 전문용어에는 쉬운 설명 병기, 목차·요약 카드 제공 | 의학 논문 수준 문장을 그대로 노출 |
| 출처가 보이는 정보 | 모든 콘텐츠에 출처·검수자·수정일 표기 | 출처 없는 통계/주장 |
| 광고와 정보의 명확한 구분 | 상품 추천·제휴 링크에 항상 표시 배지 | 네이티브 광고를 일반 콘텐츠처럼 배치 |
| 빠르고 정교한 검색 | 오타 보정, 동의어, 자연어 질문 지원 | 정확한 키워드가 아니면 결과 없음 |
| 안전한 구매 | 판매 유형별 정책(배송/환불/취소) 명시 | 정책 미표시 상태로 결제 진행 허용 |
| 과장되지 않은 상품 연결 | 콘텐츠 맥락에 맞는 상품만 추천 | 관련 없는 상품을 본문에 강제 삽입 |

## 2. 차별점 (최소 5개)

1. **양방향 연결**: 건강 콘텐츠 ↔ 상품 데이터가 서로를 참조하는 관계형 모델 (콘텐츠에서 상품으로, 상품 상세에서 관련 콘텐츠로).
2. **강한 브랜드 경험**: 2026 비주얼 키워드를 실제 프로덕션 품질로 구현 (경쟁 헬스 정보 사이트 대비 시각적 임팩트 우위).
3. **검색의 이중 경로**: 키워드/카테고리 탐색 + 자연어 질문(AI)의 병렬 지원.
4. **완전한 멀티커머스 수용**: 자체/제휴/위탁/입점/디지털/구독/예약 7종 판매 유형을 하나의 데이터 모델로 수용.
5. **영역별 절제된 UX**: 정보 본문에서는 신뢰와 집중을, 브랜드/커머스 영역에서는 화려함을 선택적으로 적용하는 이원화 전략.

## 3. 사용자 유형별 가치

- **정보 탐색형 사용자**: 증상/질환을 빠르게 이해하고 응급 여부를 즉시 파악.
- **구매 결정형 사용자**: 콘텐츠에서 검증된 맥락으로 상품을 비교하고 구매.
- **관리형 사용자(만성질환/시니어 등)**: 구독형 상품, 예약형 서비스로 반복적인 관리 지원.
- **탐색형 사용자**: 최신 건강 이슈, 계절별 콘텐츠로 발견형 탐색.

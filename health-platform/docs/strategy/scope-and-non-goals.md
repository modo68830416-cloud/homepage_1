# 범위와 비목표 (Scope & Non-Goals)

- 작업 ID: TASK-001

## 1. 이번 단계(TASK-001~010)의 범위

- 건강정보 콘텐츠 열람 경험 (증상/질환/생활관리 등)
- 통합 검색 및 자연어 질문(AI 발견 경험, 진단 아님)
- 멀티커머스 상품 도메인 (자체/제휴/위탁/입점/디지털/구독/예약)
- 관리자 CMS 및 운영 도구
- 디자인 시스템, 모션/3D 시스템
- 성능/보안/SEO/접근성 기준
- QA/배포/CI 파이프라인의 뼈대

## 2. 비목표 (Non-Goals)

- **실제 결제사 연동(PG 실연동)**: 결제 어댑터 인터페이스와 목(mock) 구현까지만 다룬다. 실제 PG사(토스페이먼츠, 이니시스 등) 계약/연동은 범위 밖이다. → [결정 필요] 실제 PG 선정
- **실제 의료진 감수 프로세스 운영**: 검수 워크플로 *구조*는 구축하되, 실제 의료 전문가 섭외 및 감수 계약은 범위 밖이다.
- **실제 물류/배송사 연동**: 배송 정책 데이터 모델까지만 다루고 실제 택배사 API 연동은 범위 밖이다.
- **개인화 추천 ML 모델 학습**: 규칙 기반/휴리스틱 추천까지만 다루고 별도 ML 파이프라인 구축은 범위 밖이다.
- **다국어/현지화**: 1차 범위는 한국어 단일 언어이다. → [결정 필요] 글로벌 확장 여부
- **네이티브 앱**: 반응형 웹만 다루며 iOS/Android 네이티브 앱은 범위 밖이다.
- **실제 서버리스/인프라 프로비저닝**: CI/CD 워크플로 파일과 런북까지 작성하되, 실제 클라우드 계정/도메인 구매·발급은 사용자 액션이 필요한 영역으로 남긴다.

## 3. 기존 프로젝트와의 관계

이 플랫폼은 저장소 내 기존 MERION/MOMO 홈페이지 프로젝트와 완전히 독립된 별도 디렉터리(`health-platform/`)에서 개발한다. 코드, 이미지 자산, 디자인 토큰을 공유하지 않는다.

## 4. 단계적 접근

TASK-001~004는 문서/디자인 시스템/모션 시스템(스캐폴딩)까지만 다루며 실 서비스 페이지 구현은 TASK-005부터 시작한다. 각 작업은 이전 작업의 산출물을 기준으로 진행한다(TASK-010 통합 순서 참고).

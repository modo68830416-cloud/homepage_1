## 2. 항목별 상세 분석 (발췌)

### 첫 화면 임팩트
- **차용**: Apple·Oura·Nike의 "하나의 오브젝트 + 최소 텍스트 + 즉시 스크롤 유도" 구조는 Hero에 적용 가치가 높다.
- **피해야 할 것**: WebMD류의 광고 밀집형 첫 화면은 신뢰도를 낮춘다. 절대 차용하지 않는다.

### 헤더 구조 · 검색 진입 방식
- Healthline/Cleveland Clinic은 검색을 헤더 우측 아이콘으로 축소해 검색 우선순위가 낮다. 본 플랫폼은 반대로 검색을 Hero 중앙에 배치해 최우선 진입점으로 삼는다(TASK-002 원칙과 연결).
- Stripe의 메가메뉴는 카테고리별 아이콘+설명 조합으로 정보량이 많아도 스캔이 쉽다. → 데스크톱 메가메뉴에 차용.

### 스크롤 연출 · 3D/영상
- Apple의 스크롤 연동 3D는 성능 위험이 크지만 자산을 구간별로 나누고 저사양 기기에서 정적 이미지로 대체하는 전략을 사용한다. → TASK-004 성능 원칙에 그대로 채택.
- Calm의 배경 영상 히어로는 모바일 데이터 사용량 이슈가 있다. → 모바일에서는 정적/저용량 루프 영상으로 대체.

### 타이포그래피 · 색상 대비
- Notion·Stripe는 본문 대비율이 매우 높고 (WCAG AA 이상) 브랜드 영역에서만 컬러를 확장한다. → 정보 영역/브랜드 영역 이원화 전략의 근거로 채택.

### 카드 시스템 · 상품 상세 경험
- Oura·Whoop·Nike는 카드보다 "무대형(스테이지) 프레젠테이션"으로 상품을 단독 조명한다. → 상품 상세 Hero에 채택.
- Airbnb의 카드형 그리드는 다량의 항목을 스캔하는 데 강하다. → 상품 목록/콘텐츠 목록에 채택.

### 콘텐츠 상세 경험
- Cleveland Clinic·Mayo Clinic은 목차 고정, 검수자 배지, 수정일 상단 노출이 강점이다. → 그대로 채택.
- 피해야 할 것: WebMD의 콘텐츠 중간 삽입 광고 배너는 신뢰를 해친다.

### 모바일 전환
- Airbnb·Nike·Oura는 모바일 전용 내비게이션(하단 탭 또는 전체 화면 오버레이 메뉴)을 별도로 설계한다. → TASK-002 모바일 내비게이션 원칙에 채택.

### 접근성
- Apple·Notion·Stripe는 포커스 링, 대체 텍스트, 시맨틱 랜드마크가 상대적으로 잘 관리되어 있다. → 접근성 기준선으로 참고.

### 성능 위험
- 3D/영상 중심 사이트(Apple, Oura, Nike, Calm) 공통 위험: 초기 로드 JS/자산 크기, LCP 지연, 저사양 기기 프레임 드랍. → TASK-004/009 성능 예산으로 대응.

## 3. 차용 가능한 원칙 (10개 이상, 디자인 판단 기준)

1. Hero는 하나의 오브젝트 + 최소 텍스트로 3초 내 메시지 전달 (Apple/Oura).
2. 검색을 Hero 중앙에 배치해 최우선 진입점으로 만든다 (자체 원칙, 경쟁사 대비 차별화).
3. 메가메뉴는 아이콘+설명 조합으로 스캔성을 높인다 (Stripe).
4. 상품 상세는 카드가 아닌 무대형 프레젠테이션을 사용한다 (Oura/Whoop/Nike).
5. 콘텐츠 상세는 검수자·수정일을 상단 고정 배지로 노출한다 (Mayo/Cleveland Clinic).
6. 브랜드 영역과 정보 영역의 색상 대비 전략을 분리한다 (Notion/Stripe).
7. 3D/영상 자산은 구간 분할 후 저사양 기기에 정적 대체 이미지를 제공한다 (Apple).
8. 모바일은 축소판이 아닌 전용 내비게이션을 설계한다 (Airbnb/Nike).
9. 카드형 그리드는 목록형 탐색(상품/콘텐츠 목록)에, 무대형은 상세 페이지에 사용한다.
10. 통계/숫자는 스크롤 트리거 애니메이션으로 강조하되 본문 근처에서는 사용하지 않는다 (Apple 제품 스펙 섹션).
11. 광고/제휴 콘텐츠는 시각적으로 구분되는 라벨을 항상 부착한다 (Healthline 대비 WebMD 반면교사).

## 4. 피해야 할 요소

- 광고 밀집형 레이아웃 (WebMD 패턴)
- 콘텐츠 본문 중간 삽입 배너 광고
- 모바일에서 데스크톱 3D를 그대로 축소 적용 (프레임 드랍 위험)
- 검색을 보조 기능으로 취급하는 헤더 구조
- 판매 유형(제휴/위탁 등) 미표시

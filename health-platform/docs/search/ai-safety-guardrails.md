# AI 검색 응답 안전 가드레일

- 작업 ID: TASK-006
- 구현: `src/app/api/ask/route.ts`, `src/lib/search/ask.ts`

## 1. 원칙

**AI는 진단하지 않는다.** `/ask`의 응답은 아래 8개 섹션 구조를 벗어날 수 없으며, 이 구조를 우회하는 자유 형식 생성 텍스트를 허용하지 않는다.

1. 질문 요약
2. 관련 일반정보
3. 관련 증상 또는 주제
4. 응급 신호
5. 관련 콘텐츠
6. 생활관리 정보
7. 관련 상품
8. 의료전문가 상담 안내 + 근거/출처 링크

## 2. 금지 표현

- 진단 확정형 문장("당신은 ~병입니다") 금지
- 처방/용량 지시 금지("~을 하루 3번 복용하세요")
- 특정 병원/특정 의사 추천 금지(이해상충 방지)
- 응급 신호가 감지된 질문에 대해 상품/구매 유도를 우선 노출하는 순서 금지 — 응급 신호 섹션은 항상 관련 상품 섹션보다 먼저 렌더링된다.

## 3. 응급 신호 감지

`src/lib/search/ask.ts`의 `EMERGENCY_KEYWORDS` 목록에 매칭되는 질문은 응급 신호 섹션을 최상단(관련 상품보다 먼저)에 강제 배치하고, 배지 색상은 `--color-danger-500`을 사용한다. 이 목록은 완전하지 않으므로 → [결정 필요] 의료 자문단의 응급 키워드 목록 검수.

## 4. 근거 표시

모든 응답은 최소 1개 이상의 출처(내부 콘텐츠 또는 외부 학회 자료)를 포함해야 하며, 출처가 없는 경우 "이 주제에 대한 검수된 정보가 아직 없습니다"로 명확히 표시하고 추측성 답변을 생성하지 않는다.

## 5. 로깅과 재검토

- 모든 질문/응답은 `logAskQuery()`로 기록되며 개인 식별 정보(이름, 연락처 등)는 저장하지 않는다.
- 응급 신호로 분류된 질문은 별도 큐에 표시되어 운영자가 주기적으로 패턴을 검토할 수 있어야 한다 (TASK-008 검색 분석과 연결). → [결정 필요] 검토 주기 및 담당자.

## 6. 현재 구현의 한계 (명시적 고지)

이번 범위의 `/ask` 구현은 실제 LLM 호출이 아니라 규칙 기반(rule-based) 매칭으로 위 8단계 구조를 채우는 **스캐폴딩**이다. 실제 LLM 연동 시에도 이 문서의 구조적 제약(8섹션, 응급 우선순위, 금지 표현)은 시스템 프롬프트 수준에서 강제되어야 하며, 모델 출력이 이 구조를 벗어나면 후처리 단계에서 재구성하거나 차단해야 한다.

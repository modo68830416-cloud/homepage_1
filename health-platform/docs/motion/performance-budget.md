# 모션 성능 예산

- 작업 ID: TASK-004
- 전사 성능 예산(LCP/INP/CLS)은 TASK-009에서 다룬다. 이 문서는 모션/3D 전용 예산이다.

## 1. 원칙

- 본문(콘텐츠 읽기 영역)과 결제 페이지에는 무거운 3D를 사용하지 않는다.
- 3D 자산(모델, 텍스처)은 항상 지연 로딩(`next/dynamic`, `ssr: false`)한다.
- 모바일 저사양 기기에서는 3D 대신 정적 대체 이미지를 제공한다.
- FPS가 지속적으로 저하되면 효과를 자동 축소한다(픽셀비 다운스케일 → 정적 이미지 순).
- `prefers-reduced-motion` 요청 시 장식성 모션을 제거한다.
- LCP 요소(Hero 헤드라인, 첫 화면 대표 이미지)에는 무거운 애니메이션을 직접 적용하지 않는다 — 애니메이션은 LCP 요소의 부모/배경에만 적용하고 LCP 텍스트 자체는 즉시 렌더링한다.

## 2. 예산 표

| 자산 유형 | 예산 | 비고 |
|---|---|---|
| Hero 3D 씬 JS(three + r3f + drei, gzip) | ≤ 180KB 추가 청크 | 메인 번들과 분리, 지연 로딩 |
| 3D 모델 지오메트리 | ≤ 1.5MB (draco 압축) | 저사양 기기는 로드 자체를 생략 |
| Hero 정적 대체 이미지 | ≤ 200KB (AVIF/WebP) | LCP 후보이므로 우선순위 힌트 부여 |
| 초당 프레임(FPS) 최소 기준 | 45fps | 2초 이상 미만 지속 시 자동 축소 |
| 자동 축소 1단계 | pixelRatio를 1로 고정 | |
| 자동 축소 2단계 | 3D 캔버스 언마운트 → 정적 이미지 대체 | |

## 3. 기기 판정 휴리스틱 (`useDeviceCapability`)

- `navigator.hardwareConcurrency < 4` → 저사양으로 간주
- `navigator.deviceMemory && navigator.deviceMemory <= 4` → 저사양으로 간주
- User-Agent 기반 모바일 판정 + WebGL2 컨텍스트 생성 실패 → 3D 비활성화
- 위 조건에 해당하지 않아도 런타임 FPS 모니터링(§2)으로 2차 방어선을 둔다.

## 4. 실패 격리

- `HeroScene`은 자체 에러 바운더리로 감싸며, 3D 렌더링 실패 시 자동으로 정적 이미지 fallback을 렌더링하고 콘솔에만 에러를 기록한다(사용자에게 에러 UI 노출 금지 — 핵심 기능 영향 없음 원칙).

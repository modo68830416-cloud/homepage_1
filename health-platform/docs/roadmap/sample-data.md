# 샘플 데이터

- 작업 ID: TASK-010 산출물

이번 스캐폴딩 단계는 별도 시드 스크립트 대신, 실행 가능한 애플리케이션 코드 자체에 샘플 데이터를 내장했다(데이터베이스가 없기 때문). 아래 파일들이 "샘플 데이터" 산출물이다.

| 파일 | 내용 |
|---|---|
| `src/lib/content/data.ts` | 건강정보 콘텐츠 7건(1건은 DRAFT 상태로 승인 게이트 데모용), 카테고리 7종 |
| `src/lib/products/data.ts` | 판매유형 7종을 각 1건씩 포함한 상품 7건, 스토어 카테고리 10종 |
| `src/lib/admin/roles.ts` | 관리자 역할 9종 및 섹션 권한 매트릭스 |
| `src/app/admin/users/page.tsx` | 운영진 목업 사용자 9명 |

실제 DB 도입 시 이 파일들의 배열 리터럴을 시드 스크립트(예: `prisma db seed`)로 이전하고, 조회 함수(`getArticleBySlug` 등)의 시그니처는 그대로 유지하면 나머지 애플리케이션 코드를 수정하지 않아도 된다([[dependency-map]] §2 참조).

# 접근성 규칙 (디자인 시스템 레벨)

- 작업 ID: TASK-003
- 전사 접근성 정책(자동화 테스트 포함)은 TASK-009에서 다시 다룬다. 이 문서는 디자인 시스템이 준수해야 할 기준선을 정의한다.

## 1. 색상 대비

- 본문/UI 텍스트: WCAG AA(4.5:1) 이상. 응급/경고 정보: AAA(7:1) 지향 ([[color-system]] §3).
- 색상만으로 의미를 전달하지 않는다(판매유형 배지는 아이콘+텍스트 병행).

## 2. 모션

- `prefers-reduced-motion: reduce` 감지 시 모든 장식성 모션(패럴랙스, 3D 부유, 자동 재생 루프)을 제거하거나 정적 상태로 대체한다 ([[motion-tokens]] §5).
- 자동재생 캐러셀/영상은 일시정지 컨트롤을 항상 제공한다.

## 3. 키보드 탐색

- 모든 인터랙티브 요소는 Tab 순서로 도달 가능해야 하며, 시각적으로 숨겨진 장식 요소는 `tabindex="-1"` 또는 `aria-hidden`으로 명확히 제외한다.
- 포커스 표시는 배경색과 무관하게 항상 보이는 2px 이상 아웃라인(`--border-focus`)을 사용한다. `outline: none`으로 제거하고 대체하지 않는 것을 금지한다.
- 메가메뉴/모바일 오버레이는 포커스 트랩을 구현하고 `Esc`로 닫을 수 있어야 한다.

## 4. 스크린리더 / 시맨틱

- 랜드마크(`header`, `nav`, `main`, `footer`, `aside`)를 페이지마다 명확히 구성한다.
- 이미지는 의미 있는 경우 대체 텍스트, 장식용인 경우 빈 `alt=""`.
- 응급 안내(`EmergencyBanner`)는 `role="alert"`로 스크린리더에 즉시 안내되어야 한다.

## 5. 확대/축소 및 반응형

- 200% 브라우저 확대에서도 가로 스크롤 없이 콘텐츠가 재배치되어야 한다(rem 기반 단위, [[typography]] §6).
- 터치 타겟 최소 44x44px ([[mobile-navigation]] §5).

## 6. 폼 오류 안내

- 오류 필드는 시각적 표시(테두리 색상) + 텍스트 메시지 + `aria-describedby` 연결을 모두 갖춘다.

## 7. 3D/모션 실패 시 대체

- 3D 캔버스 로드 실패 또는 저사양 기기 판정 시 정적 이미지로 즉시 대체하며, 이 대체 이미지도 동일한 접근성 기준(대체 텍스트 등)을 만족해야 한다.

# 색상 시스템

- 작업 ID: TASK-003
- 원시 토큰 값: [[design-tokens]] §1

## 1. 색상 역할 3단 구조

1. **Primitive(원시)**: `--color-brand-600`처럼 실제 값을 가진 토큰. 컴포넌트에서 직접 참조하지 않는다.
2. **Semantic(의미)**: `--text-primary`, `--bg-surface`처럼 역할을 가진 토큰. 컴포넌트는 이 레이어만 참조한다.
3. **Component(컴포넌트)**: 특정 컴포넌트 전용 예외 토큰(예: `--badge-affiliate-bg`)이 필요할 때만 최소한으로 추가한다.

## 2. 영역별 색상 사용 규칙

| 영역 | 배경 | 텍스트 | 강조 |
|---|---|---|---|
| 브랜드 | `--color-brand-950`→`--color-brand-600` 그라데이션 | `#ffffff` (고정 흰색, 대비 확보) | `--color-accent-500` |
| 정보 | `--color-surface-0` | `--color-ink-900` / `--color-ink-700` | `--color-info-500` (링크) |
| 커머스 | `--color-surface-50` | `--color-ink-900` | 판매유형별 배지 색상([[design-tokens]] 참조) |

## 3. 명도 대비 기준

- 본문 텍스트 vs 배경: 최소 WCAG AA(4.5:1) 이상, 데이터/가격 텍스트도 동일 기준 적용.
- 큰 제목(24px/700 이상): 최소 3:1 허용(WCAG 대형 텍스트 기준)이나 가능한 4.5:1을 지향.
- 브랜드 영역 흰색 텍스트는 그라데이션 배경의 가장 밝은 지점 기준으로도 4.5:1을 만족하도록 오버레이(반투명 어두운 레이어)를 텍스트 뒤에 추가한다.
- 응급/경고 색상(`--color-danger-500`, `--color-emergency-bg`)은 별도로 7:1(WCAG AAA)을 목표로 한다 — 의료 안전 정보이므로 가장 엄격한 기준 적용.

## 4. 판매유형 배지 원칙

- 배지 색상은 텍스트가 아닌 배경+아이콘 조합으로 구분하여 색각 이상 사용자도 유형을 구분할 수 있게 한다(색상 단독 의존 금지, [[accessibility-rules]] 연결).
- 배지 라벨 예: `자체배송`, `제휴`, `위탁배송`, `입점`, `디지털`, `구독`, `예약`.

## 5. 다크 스코프 (브랜드 영역 한정)

`data-theme="dark"`가 적용된 브랜드 섹션 내부에서만:

```
--bg-page: var(--color-brand-950);
--text-primary: #ffffff;
--text-secondary: var(--color-brand-400);
```

정보/커머스 영역에는 이 스코프를 적용하지 않는다(§8 design-tokens 테마 전략과 동일 원칙).

## 6. 금지 패턴

- 순수 장식 목적의 무지개색 그라데이션 텍스트를 본문에 사용하지 않는다.
- 상태색(success/warning/danger)을 브랜드 강조색으로 전용하지 않는다(의미 혼동 방지).

# 검색 택소노미

- 작업 ID: TASK-002
- 검색 기능 상세 구현은 TASK-006에서 다룬다. 이 문서는 검색이 다뤄야 할 "분류 체계"만 정의한다.

## 1. 검색 결과 유형 (콘텐츠 유형 필터)

| 유형 코드 | 표시명 | 설명 |
|---|---|---|
| `ARTICLE` | 건강정보 | 증상/질환/생활관리 콘텐츠 |
| `PRODUCT` | 상품 | 전 판매유형 포함 |
| `CATEGORY` | 카테고리 | 분류 페이지 바로가기 |
| `FAQ` | 자주묻는질문 | 고객지원 FAQ |
| `BRAND` | 브랜드 | 브랜드 모아보기 |

## 2. 검색어 정규화 대상

- 질환명 (예: 고혈압, 당뇨병)
- 증상명 (예: 두통, 어지럼증)
- 일반어 (예: 머리아픔 → 두통)
- 전문용어 (예: 편두통, 긴장성두통)
- 약어 (예: 고지혈증 ↔ 이상지질혈증)
- 동의어 (예: 불면증 ↔ 수면장애)
- 오타 (예: 두퉁 → 두통)
- 연령·성별·상황 (예: 임산부, 소아, 갱년기)
- 상품명 / 브랜드 / 상품 특성 (예: 무설탕, 저자극)

## 3. 검색 결과 없음 처리 분류

| 상황 | 대응 |
|---|---|
| 오타로 추정 | 보정된 검색어로 자동 재검색 + "~로 검색한 결과입니다" 안내 |
| 완전히 새로운 용어 | 인기 검색어 + 카테고리 추천 노출 |
| 상품만 없음(콘텐츠는 있음) | 콘텐츠 결과만 표시 + "관련 상품 준비중" 안내 |

## 4. 검색 필터 축

- 콘텐츠 유형(§1)
- 카테고리(건강정보/스토어 트리, [[category-tree]] 참조)
- 판매유형(TASK-007 enum과 연결: DIRECT/AFFILIATE/DROPSHIP/MARKETPLACE/DIGITAL/SUBSCRIPTION/BOOKING)
- 정렬(관련도/최신순/인기순/가격순)

## 5. 자연어 질문(AI) 입력과의 관계

- 통합검색창에 자연어 문장(3어절 이상 또는 물음표 포함)이 입력되면 `/ask` 흐름으로 자동 라우팅 제안을 표시한다(강제 이동은 아님).
- AI 응답 구조는 TASK-006 §3(AI 검색 원칙)을 그대로 따른다.

## 6. 검색 로그 분류 (관리자 분석용, TASK-008 연결)

- 성공 검색 (결과 있음, 클릭 발생)
- 성공 검색 (결과 있음, 클릭 없음) → 결과 품질 재검토 대상
- 실패 검색 (결과 없음) → 콘텐츠/상품 갭 발견용
- AI 질문 로그 (별도 저장, 개인 식별 정보 최소화)

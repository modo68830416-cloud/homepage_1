# 내비게이션 모델

- 작업 ID: TASK-002
- 노드 정의: [[category-tree]] · 모바일 전용 구조: [[mobile-navigation]]

## 1. 데스크톱 헤더 구조

```
[로고]  [건강정보 ▾]  [스토어 ▾]  [검색창(확장형)]  [건강 질문하기]  [찜] [마이페이지] [로그인/회원]
```

- 검색창은 항상 보이는 확장형 입력(클릭 시 전체너비 오버레이로 확장, TASK-004 인터랙션 카탈로그의 "검색창 확장" 참조)이다. 아이콘으로 축소하지 않는다.
- `건강정보`, `스토어`는 메가메뉴로 열리며 각각 [[category-tree]]의 하위 노드를 우선순위(`노출우선순위`) 순으로 배치한다.

## 2. 메가메뉴 레이아웃 원칙

- 한 메가메뉴 패널에 최상위 하위 카테고리는 최대 6~8개까지만 노출하고, 나머지는 "전체보기" 링크로 연결한다(한 화면 과다 노출 금지 원칙).
- 각 컬럼 상단에 대표 이미지/아이콘 1개를 배치해 스캔성을 높인다 (Stripe 벤치마크 패턴, [[visual-benchmark-matrix]] 참조).
- 건강정보 메가메뉴는 좌측에 "증상별/질환별"(탐색 진입점), 우측에 "생활관리/대상별/계절별"(맥락 탐색)을 분리 배치한다.
- 스토어 메가메뉴는 좌측에 실물 카테고리, 우측에 디지털/구독/예약(비실물)을 분리 배치해 성격이 다른 상품군을 혼동하지 않게 한다.

## 3. 사용자 상태별 우측 영역

| 상태 | 노출 노드 |
|---|---|
| GUEST | N-ACC-LOGIN, N-ACC-SIGNUP |
| MEMBER | N-ACC-WISHLIST(찜 카운트), N-ACCOUNT(마이페이지 진입) |

## 4. 콘텐츠·상품 상호 연결 내비게이션

- 콘텐츠 상세(`N-HEALTH-ARTICLE`)에는 전역 메뉴 외에 로컬 "관련 상품" 섹션이 항상 존재 (전역 IA 노드는 아니지만 필수 컴포넌트).
- 상품 상세(`N-SHOP-PRODUCT`)에는 로컬 "관련 건강정보" 섹션이 항상 존재.
- 이 양방향 연결은 메뉴 트리와 별개로 콘텐츠-상품 간 관계형 태그(카테고리/증상/질환 매핑)로 구현한다 (TASK-007 상품 모델의 "관련 콘텐츠" 필드와 연결).

## 5. 브레드크럼 규칙

- 3뎁스 이상 페이지는 브레드크럼을 표시한다: `홈 > 건강정보 > 생활관리 > 수면`.
- 브레드크럼의 각 항목은 [[category-tree]] 노드 ID와 1:1 매핑되어야 한다.

## 6. 푸터 내비게이션

- 1차: 회사 정보, 고객지원(N-SUPPORT 하위 전체), 정책(N-SUPPORT-POLICY)
- 2차: 건강정보 SEO용 사이트맵 링크(검색엔진 및 사용자 하위 카테고리 전체 노출)
- 3차: 판매자 입점 문의(입점판매 관련, TASK-007 MARKETPLACE와 연결)

## 7. 우선순위 표시 규칙

- [[category-tree]]의 `노출우선순위` 값이 낮을수록(1이 최상위) 메가메뉴/모바일 메뉴 상단에 먼저 노출한다.
- 우선순위가 동일한 노드는 알파벳/가나다 순이 아닌 관리자 CMS에서 수동 정렬 가능해야 한다(TASK-008 메인 큐레이션 기능과 연결).

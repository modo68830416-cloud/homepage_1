# 모바일 내비게이션

- 작업 ID: TASK-002
- 데스크톱 대비 원칙: 모바일 메뉴는 데스크톱 메가메뉴를 단순 축소한 형태가 아니라 별도로 최적화한다.

## 1. 상단 바 (Persistent)

```
[≡ 메뉴]     [로고]     [검색 아이콘 → 탭 시 전체화면 검색 오버레이]
```

## 2. 하단 탭 바 (Primary Navigation, 5개 고정)

| 위치 | 아이콘/라벨 | 매핑 노드 |
|---|---|---|
| 1 | 홈 | N-HOME |
| 2 | 건강정보 | N-HEALTH |
| 3 | 검색(중앙, 강조) | N-SEARCH-UNIFIED |
| 4 | 스토어 | N-SHOP |
| 5 | 마이 | N-ACCOUNT |

- 하단 탭은 스크롤 시 자동 숨김/노출하지 않는다(예측 가능성 우선, 접근성 원칙).

## 3. 전체화면 메뉴 오버레이 (`≡` 탭 시)

- 1depth: 아코디언 방식으로 건강정보 / 스토어 / 검색 / 질문하기 / 고객지원을 나열.
- 2depth: 아코디언 확장 시 [[category-tree]]의 우선순위 순으로 하위 노드 노출, 6개 초과 시 "더보기".
- 하단 고정 영역: 로그인/회원가입(GUEST) 또는 마이페이지 바로가기(MEMBER).

## 4. 전체화면 검색 오버레이

- 진입 즉시 키보드 포커스, 최근 검색어(MEMBER) + 인기 검색어를 기본 노출.
- 타이핑 시 자동완성 리스트가 즉시 갱신 (TASK-006 자동완성 API 연결).
- 결과 유형 탭(콘텐츠/상품)은 가로 스크롤 칩(chip) 형태로 상단 고정.

## 5. 제스처·터치 규칙

- 모든 터치 타겟은 최소 44x44px.
- 메가메뉴 대신 아코디언을 쓰는 이유: 좁은 화면에서 다단 컬럼 메가메뉴는 스캔 비용이 커지기 때문 (TASK-002 §2 핵심 설계 원칙 "한 화면에서 너무 많은 선택지를 노출하지 않는다"와 연결).
- 뒤로가기 제스처(스와이프)는 브라우저 기본 동작을 가로채지 않는다.

## 6. 콘텐츠/상품 상세의 모바일 전용 요소

- 콘텐츠 상세: 목차를 상단 고정 "펼치기" 칩으로 축소 제공 (데스크톱은 사이드 고정 목차).
- 상품 상세: 구매 CTA를 화면 하단 고정 바(Sticky CTA)로 항상 노출.

## 7. 완료 조건 대응

- 모바일 첫 화면에서 검색 아이콘이 항상 보이고 1탭으로 전체화면 검색에 진입 가능 (TASK-005 완료조건과 연결).

# 필수 사용자 흐름

- 작업 ID: TASK-002
- 노드 ID는 [[category-tree]] 참조

## 흐름 1: 검색 → 건강정보 → 관련상품 → 상품상세 → 구매

```
N-SEARCH-UNIFIED → N-SEARCH-HEALTH(결과 탭) → N-HEALTH-ARTICLE
  → [로컬: 관련 상품 섹션] → N-SHOP-PRODUCT → [장바구니] → [결제]
```
- 이탈 방지 포인트: 콘텐츠 상세에서 관련 상품 섹션은 스크롤 30~50% 지점과 하단에 각 1회 노출.

## 흐름 2: 상품검색 → 상품상세 → 관련 건강정보 → 구매

```
N-SEARCH-PRODUCT → N-SHOP-PRODUCT
  → [로컬: 관련 건강정보 섹션] → N-HEALTH-ARTICLE (선택적 열람)
  → 뒤로가기 또는 새 탭 → N-SHOP-PRODUCT → [장바구니] → [결제]
```
- 상품 상세 이탈 없이 관련 정보를 새 탭 또는 슬라이드 패널로 열람 가능해야 한다(구매 흐름 단절 방지).

## 흐름 3: 증상 탐색 → 응급신호 확인 → 의료기관 이용 안내

```
N-HEALTH-SYMPTOMS → [증상 상세] → [응급신호 컴포넌트 노출 여부 판단]
  → (응급 신호 해당 시) 응급 안내 배너 최상단 고정 → 의료기관 이용 안내(119/응급실 안내) → 이후 콘텐츠 열람 허용
  → (응급 아님) 일반 콘텐츠 흐름 계속
```
- 이 흐름은 판매 유도보다 항상 우선한다 (TASK-006 §6, TASK-009 §6 건강정보 안전과 직접 연결).

## 흐름 4: 콘텐츠 → 북마크 → 마이페이지

```
N-HEALTH-ARTICLE → [북마크 버튼] → (비로그인 시 로그인 유도) → N-ACC-BOOKMARKS
```

## 흐름 5: 제휴상품 클릭 → 외부몰 이동

```
N-SHOP-PRODUCT(AFFILIATE) → [가격 기준 시점 안내 + 제휴 표시 확인] → /go/[trackingId] → 외부몰
```
- 이탈 전 마지막 화면에서 반드시 "외부 사이트로 이동합니다" 확인을 표시 (신규 탭 기본).

## 흐름 6: 자체상품 → 장바구니 → 결제

```
N-SHOP-PRODUCT(DIRECT) → [장바구니 담기] → /shop/cart → /shop/checkout → 주문완료 → N-ACC-ORDERS
```

## 흐름 7: 디지털상품 → 결제 → 다운로드 또는 열람

```
N-SHOP-PRODUCT(DIGITAL) → [바로 결제 또는 장바구니] → /shop/checkout
  → 주문완료 → [디지털 권한 발급] → N-ACC-ORDERS 내 "다운로드/열람" 버튼 활성화
```
- 디지털 상품은 반품 대신 "접근 취소" 정책이 별도로 필요 (TASK-007 §4 디지털상품 UX 참고).

## 흐름 8 (추가): 구독상품 가입 → 관리

```
N-SHOP-PRODUCT(SUBSCRIPTION) → [구독 시작] → /shop/checkout
  → N-ACC-ORDERS 내 구독 카드 → [일시정지/해지/다음 결제일 확인]
```

## 흐름 9 (추가): 예약상품 예약 → 확정

```
N-SHOP-PRODUCT(BOOKING) → [일정 선택] → [예약 확정] → N-ACC-ORDERS 내 예약 카드 → [취소 규정 확인 가능]
```

## 공통 원칙

- 모든 흐름에서 판매유형 배지는 상품 카드/상세/장바구니/주문내역까지 끊기지 않고 동일하게 노출된다.
- 결제로 이어지는 모든 흐름은 로그인 없이 비회원 진행이 최소 1개 이상(예: DIRECT) 가능해야 한다. → [결정 필요] 비회원 구매 허용 판매유형 범위

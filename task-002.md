## 1. 목표

건강정보, 상품, 검색, 회원, 고객지원이 복잡해져도 사용자가 길을 잃지 않는 전체 정보구조를 설계한다.

## 2. 핵심 설계 원칙

- 메인 검색을 최우선 진입점으로 둔다.
- 질환, 증상, 생활관리, 대상별, 상품별 탐색을 병렬 지원한다.
- 콘텐츠와 상품은 분리된 영역을 유지하면서 서로 연결한다.
- 데스크톱 메가메뉴와 모바일 계층 메뉴를 별도로 최적화한다.
- 한 화면에서 너무 많은 선택지를 노출하지 않는다.

## 5. URL 정책 예시

- `/health`
- `/health/symptoms`
- `/health/symptoms/headache`
- `/health/conditions/hypertension`
- `/health/lifestyle/sleep`
- `/shop`
- `/shop/products/[slug]`
- `/search?q=`
- `/ask`
- `/account/orders`
- `/support`

URL은 한글 표시가 가능하더라도 내부 슬러그는 안정적인 영문 소문자 및 하이픈을 기본으로 한다.

## 6. 필수 사용자 흐름

- 검색 → 건강정보 → 관련상품 → 상품상세 → 구매
- 상품검색 → 상품상세 → 관련 건강정보 → 구매
- 증상 탐색 → 응급신호 확인 → 의료기관 이용 안내
- 콘텐츠 → 북마크 → 마이페이지
- 제휴상품 클릭 → 외부몰 이동
- 자체상품 → 장바구니 → 결제
- 디지털상품 → 결제 → 다운로드 또는 열람

## 7. 완료 조건

- 3단계 이내에 주요 콘텐츠에 접근할 수 있다.
- 모바일 메뉴가 데스크톱 메뉴를 단순 축소한 형태가 아니다.
- 콘텐츠와 상품 카테고리가 명확히 구분된다.
- 검색 결과의 콘텐츠 유형 필터 구조가 정의되어 있다.
- URL 충돌 및 중복 콘텐츠 처리 정책이 있다.

## 8. Claude Code 실행 지시

코드 작성 없이 IA 문서만 생성한다. 모든 메뉴 노드는 고유 ID, 표시명, URL, 부모 ID, 노출 우선순위, 사용자 유형을 포함한다.

## 1. 목표

판매 방식을 하나로 고정하지 않고 여러 모델을 동시에 수용하는 상품 도메인과 사용자 경험을 구축한다.

## 2. 지원 판매 유형

- `DIRECT`: 자체 재고·자체 결제·자체 배송
- `AFFILIATE`: 외부 판매처 이동
- `DROPSHIP`: 공급사 배송
- `MARKETPLACE`: 입점 판매자 상품
- `DIGITAL`: 파일·영상·프로그램
- `SUBSCRIPTION`: 정기 결제
- `BOOKING`: 예약형 상품 또는 서비스

## 4. 판매유형별 UX

### 자체판매

장바구니, 결제, 배송, 취소, 반품을 제공한다.

### 제휴판매

가격 기준 시점, 외부몰 이동 안내, 제휴 표시를 제공한다.

### 위탁판매

공급사별 배송비와 묶음배송 불가 여부를 표시한다.

### 입점판매

판매자 정보, 판매자별 배송·반품 정책을 표시한다.

### 디지털상품

다운로드 횟수, 접근 기간, 라이선스를 관리한다.

### 구독

결제 주기, 다음 결제일, 일시정지, 해지를 제공한다.

### 예약

일정 선택, 예약 확정, 취소 규정을 제공한다.

## 6. 완료 조건

- 상품 데이터 모델을 변경하지 않고 판매 유형을 추가할 수 있다.
- 판매 유형별로 불필요한 필드는 조건부 표시된다.
- 한 장바구니에 넣을 수 없는 판매 유형 정책이 정의되어 있다.
- 외부 이동과 내부 결제가 명확히 구분된다.
- 광고·제휴 고지가 누락되지 않는다.

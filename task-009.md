## 1. 목표

화려한 디자인 때문에 속도, 접근성, 검색 노출, 보안, 건강정보 신뢰가 희생되지 않도록 출시 기준을 구축한다.

## 2. 성능 예산

권장 목표:

- LCP: 2.5초 이하
- INP: 200ms 이하
- CLS: 0.1 이하
- 초기 JS: 페이지 유형별 예산 설정
- Hero 영상과 3D는 지연 및 적응형 로딩
- 모바일 저사양 대체자산 제공

## 3. 보안

- 인증·인가
- 관리자 MFA 검토
- 입력값 검증
- XSS·CSRF·SQL Injection 방어
- 업로드 파일 검사
- 결제 위변조 방지
- Rate Limit
- 감사 로그
- 비밀정보 관리
- 개인정보 최소 수집
- 백업·복구

## 4. SEO

- 콘텐츠·상품 구조화 데이터
- Sitemap 분리
- Canonical
- robots 정책
- Open Graph
- 품절·삭제 상품 정책
- 카테고리 색인 정책
- 내부 링크
- 작성자·검수자·수정일
- 검색 결과 페이지 색인 차단 정책

## 5. 접근성

- WCAG 기반 점검
- 키보드 탐색
- 포커스 표시
- 색상 대비
- 대체 텍스트
- 폼 오류 안내
- 모션 감소
- 200% 확대
- 스크린리더 랜드마크
- 터치 영역

- 출처

- 과장 표현 필터
- 사용자 신고
- 오래된 콘텐츠 재검수 알림

## 7. 산출물

## 8. 완료 조건

- 화려한 메인 화면도 성능 예산을 넘지 않는다.
- JavaScript 비활성 또는 3D 실패 시 핵심 콘텐츠가 보인다.
- 결제와 개인정보 영역에 불필요한 추적 스크립트가 없다.
- 모든 주요 페이지가 접근성 자동 검사와 수동 검수 절차를 갖는다.
